md # Branch Creation Protocol ## Purpose This document defines how new AI branches are created inside the development system. ## Core Principle New branches must be created in a controlled way so the repository structure remains stable and understandable. ## Required Actions For New Branch Creation When a new branch is created, the following must be done: 1. Create a branch memory file in: /docs/branches 2. Register the branch in: /docs/registry/branch-registry.md 3. Define the branch status: - active - planned - paused - archived 4. Ensure the branch has a clear responsibility and scope. ## Branch Memory Requirement Each branch must have its own memory file. Recommended format: /docs/branches/[branch-name].md ## Governance Rule No branch should be created without being reflected in the registry. ## Practical Principle A branch exists only when: - its memory file exists - its role is defined - its registry entry exists
